# /quick-review — システム簡易レビュー（直近2日間）

以下のステップでシステムの簡易レビューを実行してください。

## ステップ1: レビュースクリプトを実行する

```bash
python system_review.py --mode quick
```

## ステップ2: 結果を解釈して提示する

出力を読んで、以下の形式でユーザーに報告してください：

**🔴 今すぐ対応が必要な問題（CRITICAL）**
- CRITICAL の問題を箇条書きで要約する
- 問題がない場合は「CRITICALなし ✅」と示す

**🟡 近日中に対応すべき問題（WARNING）**
- WARNING の問題を箇条書きで要約する
- 問題がない場合は「WARNINGなし ✅」と示す

**修正プロンプト**
- レポート内に「# 修正依頼:」セクションがあれば、そのまま提示する
- 問題がなければ「修正は不要です」と伝える

## ステップ3: 次のアクションを提案する

- CRITICALがある場合 → 「/full-review で詳細を確認することを推奨します」と伝える
- 問題がない場合 → 「システムは正常です。次回は /full-review で精度監査も実施できます」と伝える

---
*対象: 直近2日間のデータ鮮度・航路カバレッジ・スクレイプ異常を高速チェック*
*精度監査（FN/FP分析）を含む全監査は `/full-review` を使用してください*
