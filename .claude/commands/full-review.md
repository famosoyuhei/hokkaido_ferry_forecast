# /full-review — システム総合レビュー（直近14日間・精度監査込み）

以下のステップでシステムの総合レビューを実行してください。

## ステップ1: レビュースクリプトを実行する

```bash
python system_review.py --mode full
```

実行には30秒〜1分かかる場合があります。

## ステップ2: 結果を解釈して報告する

出力を読んで、以下の形式でユーザーに報告してください：

**📊 総合ステータス**
- CRITICAL / WARNING / INFO の件数を示す
- 予報精度サマリ（精度%・FN件数・FP件数）を示す

**🔴 今すぐ対応が必要な問題（CRITICAL）**
- CRITICAL の問題を1問題1段落で要約する
- 担当AI社員名（海上気象予報取得AI / 実測取得AI / フェリー運航記録取得AI / 予報精度監査AI）を明記する

**🟡 近日中に対応すべき問題（WARNING）**
- WARNING の問題を箇条書きで要約する

**ℹ️ 参考情報（INFO）**
- INFO を簡潔に列挙する

**修正プロンプト（すぐに使える形式）**
- レポート内に「# 修正依頼:」セクションがあれば、そのまま提示する
- これをそのまま新しい Claude Code セッションに貼り付けることで修正を依頼できることを説明する

## ステップ3: 対話的フォローアップ

ユーザーが以下のような質問をした場合に対応してください：
- 「この問題を今すぐ直して」→ 修正を実施する
- 「FNが多い原因は何？」→ accuracy_stats の詳細を分析して説明する
- 「実測データを補完して」→ `python actual_weather_collector.py` の実行を提案する
- 「もう一度チェックして」→ `python system_review.py --mode quick` を再実行する

---
*対象: 直近14日間の全AI社員による総合監査*
*含む内容: データ鮮度・航路カバレッジ・スクレイプ品質・FN/FP精度分析・修正プロンプト自動生成*
*日次簡易チェックには `/quick-review` を使用してください*
