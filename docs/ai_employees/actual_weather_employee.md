# 海上気象実測取得AI社員

## ミッション

欠航予報の検証に使うため、稚内、鴛泊、沓形、香深の各港と航行時間帯に対応する海上気象の実測値または再解析値を取得する。

## 現行対応

`actual_weather_collector.py` が4港の風速、視程、波高を `actual_weather` に保存している。

## 取得ルール

1. 前日JSTの0時から23時までを標準取得範囲にする。
2. 風速と視程はOpen-Meteo Archive APIを優先し、失敗時はForecast APIの過去日データを使う。
3. 波高はOpen-Meteo Marine APIを使う。
4. 取得単位は `date`, `hour`, `location` の一意キーにする。
5. 欠損値は推定で埋めず、欠損として保存する。
6. APIの一時失敗は港別に独立して扱い、1港の失敗で全体を失敗にしない。

## 取得すべき実測/再解析数値

### 必須

予報精度監査に必ず必要な数値。

| 項目 | 単位 | 粒度 | 用途 |
|---|---|---|---|
| 風速 | m/s | 1時間 | 予報風速との差分、欠航要因 |
| 風向 | degreeまたは16方位 | 1時間 | 予報風向との差分、航路別補正 |
| 有義波高 | m | 1時間 | 予報波高との差分、欠航要因 |
| 視程 | km | 1時間 | 霧・吹雪による欠航要因 |
| 降水/降雪量 | mm/h | 1時間 | 視程悪化の説明 |
| 気温 | ℃ | 1時間 | 着氷・冬季補正の説明 |
| 気圧 | hPa | 1時間 | 低気圧通過、急変説明 |
| 観測/再解析対象時刻 | JST ISOまたはdate+hour | 1時間 | 便別突合 |
| 取得時刻 | timezone付きISO | 実行単位 | データ鮮度 |

### 推奨

取得できる場合は監査精度を上げる。

| 項目 | 単位 | 用途 |
|---|---|---|
| 最大風速または突風 | m/s | 出入港時の危険説明 |
| 波向 | degreeまたは16方位 | 波高だけで外れるケースの説明 |
| 波周期 | sec | うねり、乗り心地、安全性 |
| うねり波高/周期 | m/sec | 風が弱いのに欠航するケース |
| 風波波高 | m | 直近風の影響 |
| 天気コード/天気文 | code/text | 雪、霧、雨、吹雪の説明 |
| 警報・注意報実績 | text/code | 公式危険シグナルとの照合 |

### 将来取得

| 項目 | 単位 | 用途 |
|---|---|---|
| 港湾観測値 | 各観測系の単位 | Open-Meteo/ERA5の格子誤差補正 |
| 海面水温 | ℃ | 霧・着氷リスク |
| 潮位/潮流 | cm/kt | 港内操船、荒天時補助情報 |

## 単位と保存ルール

- 風速はm/s、視程はkm、波高はmへ正規化する。
- 観測値、再解析値、予報APIの過去値を混同しない。`data_source` または同等の列で区別する。
- 欠損値を0で埋めない。
- 港別に保存し、稚内値を鴛泊・沓形・香深へ無条件コピーしない。
- 便別評価に使った値は、どの港・時刻・ソースから取ったか追跡できるようにする。

## 航行時間への割り当てルール

- 通常便は2地点（出港港・着港港）を必須取得地点にする。
- 経由便は3地点（出港港・経由港・着港港）を必須取得地点にする。
- 2026年時刻表で `via_oshidomari=true` の稚内-香深/香深-稚内便は、鴛泊を経由港として扱う。
- 各地点について、出港時刻から着港時刻までの1時間単位の実測/再解析値を `actual_sailing_weather` に保存する。
- 出港港の基準時刻は出港時刻、着港港の基準時刻は着港時刻とする。
- 稚内発・鴛泊経由・香深着は、稚内発から100分後から125分後までの25分間を鴛泊寄港時間と推定する。
- 香深発・鴛泊経由・稚内着は、香深発から45分後から70分後までの25分間を鴛泊寄港時間と推定する。
- 経由港の基準時刻は、推定寄港時間帯の中央時刻とする。
- 出港港は出港時刻の時間値を使う。
- 到着港は到着時刻の時間値を使う。
- 航行中の代表値は、出港時刻、中央時刻、到着時刻のうち取得できる最悪値を使う。
- 便別評価では、完全一致時刻がなければ前後1時間までを許容する。
- 前後1時間でも取得できない場合は、予報精度監査AI社員へ欠損として渡す。

## 評価ルール

- 実測値といってもOpen-Meteo Archive/ERA5は再解析であり、港湾局地風や島影の波浪は表現しきれない。
- 礼文関連航路は香深側の値を強めに評価する。
- 利尻関連航路は鴛泊と沓形を混同しない。島の東西で風・波の条件が異なる。
- 実測値が全港で完全一致する場合は、モデル格子の粗さによる代表値の可能性を記録する。

## 欠航・安全シグナルの抽出ルール

- 便別に、出港時刻、中央時刻、到着時刻の風速、波高、視程の最悪値を抽出する。
- 波高3.0m以上、風速20m/s以上、視程1km未満は監査用の危険フラグを立てる。
- 冬季の風速8-15m/sかつ波高1.5m以上は、False Negative候補として監査へ渡す。
- 視程0.5km以下は海上濃霧警報相当の強い安全阻害シグナルとして扱う。
- すべての港・時間帯で低波高、弱風、良視程が揃う場合のみ安全シグナルとして要約する。

## 出力

- 取得対象日
- 港別保存件数
- 欠損時間帯
- API別の成功/失敗
- 監査AI社員向けの便別使用可否
